# История изменений (CHANGELOG)

Все заметные изменения в этом проекте будут документироваться в этом файле.

## [1.0.0] - 2026-06-22

### Добавлено
- Начальная структура интерактивного скрипта настройки системы [setup.sh](file:///Users/artur/Documents/Demo/ubuntu-auto/setup.sh).
- Автоматическое определение прав суперпользователя `sudo` / `root`.
- Поддержка TUI (терминального графического интерфейса) на базе `whiptail`.
- Прогресс-бар (`whiptail --gauge`) при установке базовых пакетов — показывает текущий пакет и процент завершения `[N/M]` по каждому элементу.
- Поэтапный прогресс-бар при установке Docker — 6 именованных шагов (очистка, зависимости, GPG, репозиторий, Engine, запуск службы) с отображением % выполнения.
- Поэтапный прогресс-бар при установке Node.js — 3 шага (подключение NodeSource, установка nodejs, финал) с отображением % выполнения.
- Функции локализации (установка русской локали `ru_RU.UTF-8`).
- Настройка часового пояса `Asia/Novokuznetsk`.
- Модули установки базового ПО: `nano`, `zip`, `git`, `openssh-server`.
- Установка контейнеризации: Docker Engine, Docker Compose с поддержкой обратной совместимости (вызов через `docker-compose`).
- Динамический опрос версий Node.js через официальный API Node.js и установка выбранной версии через репозиторий NodeSource.
- Документация в [README.md](file:///Users/artur/Documents/Demo/ubuntu-auto/README.md) с инструкцией по установке и запуску.
- Интерактивный выбор при установке SSH: пользователь может установить OpenSSH-server по умолчанию либо применить свою преднастройку (порт 22, root-вход по паролю, доступ только из подсетей 192.168.*, 10.*, 172.*, 127.*).
- Возможность выбора и установки сетевых утилит `speedtest-cli` (тест скорости интернета) и `iperf3` (тест пропускной способности сети) через меню базового ПО.
- Инструкция по установке переведена на использование `wget` по умолчанию (так как он предустановлен в чистой системе), а установка `curl` перенесена внутрь автоматических зависимостей скрипта.
- Добавлена функция настройки автоматического входа `root` для LXC-контейнеров в Proxmox (в раздел настроек сервера).
- Информационный дашборд при запуске скрипта со сводкой параметров системы (ОС, время работы, использование диска и ОЗУ, локальный и внешний IP-адреса).
- Реализована универсальная поддержка скрипта для дистрибутивов Debian GNU/Linux (адаптирована настройка русской локали и автоматическое переключение репозиториев установки Docker).
- Документация в [README.md](file:///Users/artur/Documents/Demo/ubuntu-auto/README.md) адаптирована под совместную поддержку дистрибутивов Debian и Ubuntu (требования к ОС обновлены с фиксированного списка на любые современные версии).
- Интеграция автоматической проверки установленного ПО и системных параметров:
  - Версии Docker и Node.js динамически определяются и выводятся в главном меню TUI.
  - При попытке переустановить уже установленный Docker или Node.js запрашивается явное подтверждение пользователя через диалог.
  - Наличие базового ПО (Nano, Git, SSH, Speedtest, iPerf3) и настроек (локаль, таймзона, LXC-автологин) сканируется при открытии соответствующих подменю; уже присутствующие компоненты помечаются как `(уже установлено)` и по умолчанию отключены в чекбоксе.




### Изменено
- Полностью удалено логирование в файлы (избавились от файла логов `/var/log/ubuntu-setup.log`). Вывод выполнения всех системных команд теперь перенаправлен в `/dev/null` для полной бесшумности выполнения на заднем плане TUI.
- Алгоритм динамического опроса Node.js расширен: теперь он выводит не только активные LTS версии (v24, v22, v20, v18), но и самую актуальную Current версию (v26) с официального сайта, помечая её в TUI-меню.
- Отключен жесткий режим прерывания скрипта при ошибках (`set -e`), внедрена гибкая обработка ошибок на каждом этапе установки с показом информативных окон ошибок в TUI вместо падения скрипта.
- Поведение при определении кодового имени дистрибутива в `setup_docker` изменено на fail-fast: если скрипт не может автоматически определить имя версии (например, `bookworm` или `jammy`), он аварийно завершает установку Docker с показом ошибки в TUI, чтобы избежать добавления некорректных репозиториев.
- Проект переименован из `ubuntu-auto` в `server-auto` (обновлен удаленный репозиторий Git, заменены все внутренние ссылки в файлах и брендинг скрипта в меню TUI).





### Исправлено
- Проблема с некорректным отображением кириллических символов (кракозябр `<92><8B>...`) в диалоговых окнах `whiptail` на чистых серверах за счет принудительного экспорта локали `C.UTF-8` во время выполнения скрипта.
- Пустое диалоговое окно при выборе версии Node.js (исправлен синтаксис `whiptail --radiolist` с переходом на передачу троек аргументов: тег, описание, статус выбора).
- Проблема со сменой версий Node.js на другую мажорную (теперь перед установкой новой версии старый пакет `nodejs` полностью удаляется и очищается из системы через `apt-get remove/purge/autoremove`, а подключение репозитория выполняется через официальный скрипт от NodeSource `setup_XX.x`. Это гарантирует чистую и корректную установку Node.js и npm выбранной версии).
- Упрощена функция настройки русской локали: теперь используются проверенные системные команды `apt install language-pack-ru` и `update-locale LANG=ru_RU.UTF-8`.
- Потенциальный баг и логическая ошибка определения кодового имени дистрибутива в `setup_docker` (из-за успешности команды `echo` резервная ветка через оператор ИЛИ `||` никогда не выполнялась. Код исправлен на получение переменной `VERSION_CODENAME` напрямую из `/etc/os-release`). Также добавлен динамический опрос `lsb_release -cs` в качестве резервного варианта перед применением хардкода.
- Ограничение доступа SSH (`AllowUsers`) исправлено: удалены перекрывающие диапазоны `*@10.*.*.*` и `*@172.*.*.*`, оставлена только подсеть пользователя `*@192.168.1.*` и loopback `*@127.0.0.1`.
- Логическая ошибка в перезапуске службы SSH: `&&` заменён на `||`, теперь сначала пробуется `systemctl restart ssh` (Ubuntu), при ошибке — `sshd` (Debian).
- Симлинк `docker-compose` теперь ищется в двух возможных путях: `/usr/libexec/docker/cli-plugins/` и `/usr/lib/docker/cli-plugins/`, что обеспечивает совместимость с разными дистрибутивами.
- Дублирование вызовов `apt-get update` устранено через вспомогательную функцию `run_apt_update()` с глобальным флагом `APT_UPDATED`. Обновление индекса APT теперь выполняется только один раз за сессию скрипта.
- Добавлена защита от запуска через `/bin/sh` вместо `/bin/bash`: скрипт завершается с понятной ошибкой, если `$BASH_VERSION` не определена.
- Добавлен вывод прогресса (`echo`) в функцию `install_script_deps` на этапе до инициализации `whiptail`, чтобы пользователь видел что происходит во время первоначального обновления пакетов.
- Проблема обрезания и некрасивого переноса длинного текста в TUI диалоговых окнах `whiptail` (размеры всех прогресс-баров `--gauge` увеличены с 65 до 78 символов по ширине, информационных окон `show_progress` и ошибок — до 70/72 символов, главного меню и списков выбора — до 75 символов).
