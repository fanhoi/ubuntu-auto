# Скрипт автонастройки серверов (Debian / Ubuntu)

Интерактивный Bash-скрипт с терминальным графическим интерфейсом (TUI) на базе `whiptail`, предназначенный для быстрой первоначальной настройки и установки необходимого ПО на чистых серверах под управлением ОС Debian и Ubuntu.

## Возможности скрипта

Скрипт разбит на логические модули и предоставляет удобное меню для следующих настроек:

1. **Настройка системы**:
   - Генерация и активация русской локали (`ru_RU.UTF-8`).
   - Установка часового пояса `Asia/Novokuznetsk`.
2. **Установка базового ПО**:
   - Удобный текстовый редактор `nano`.
   - Утилиты для работы с архивами `zip` и `unzip`.
   - Система контроля версий `git`.
   - SSH-сервер `openssh-server`.
3. **Установка контейнеризации**:
   - Docker Engine, Docker CLI, Buildx.
   - Плагин Docker Compose.
   - Создание символической ссылки `/usr/local/bin/docker-compose` для обратной совместимости.
   - Добавление текущего пользователя в группу `docker`.
4. **Установка Node.js**:
   - Динамический запрос доступных LTS-версий с официального сайта Node.js.
   - Установка выбранной мажорной версии (например, v18, v20, v22) через официальный репозиторий NodeSource.

## Запуск скрипта

### Вариант 1. Локальный запуск (если репозиторий клонирован)

Перейдите в папку с проектом, сделайте скрипт исполняемым и запустите его:

```bash
chmod +x setup.sh
./setup.sh
```

### Вариант 2. Запуск одной командой (прямо из GitHub)

Когда вы опубликуете скрипт на GitHub, вы сможете запускать его на любом чистом сервере без предварительного скачивания всего репозитория:

```bash
bash -c "$(wget -qO- https://raw.githubusercontent.com/fanhoi/ubuntu-auto/main/setup.sh)"
```

## Требования

- ОС: Debian / Ubuntu (любые современные версии)
- Пользователь с правами `sudo` или `root`
- Наличие интернет-соединения
